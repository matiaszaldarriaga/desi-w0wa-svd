# Universal distance modes from DESI BAO and Type Ia supernovae

Companion code and data for the paper "Universal distance modes from DESI BAO
and Type Ia supernovae: what do cosmological rulers actually measure?" by Matias
Zaldarriaga (Institute for Advanced Study, Princeton).

## Summary

All low-redshift distance probes -- DESI BAO and three Type Ia supernova
compilations -- measure the same leading SVD mode c_0, which is a near-perfect
proxy for Omega_m h^2 in LCDM chains (R^2 > 0.99). In the w0-wa extension, a
second mode c_1 becomes measurable, but the BAO c_1 tension with LCDM is only
+1.2 sigma. The DESI w0-wa preference is dominated by c_0, not by the dark
energy direction. The absence of SN confirmation or detection of a deviation in
the c_1 direction lends credence to the hypothesis that the tension originates
in the CMB--BAO calibration rather than in new low-redshift physics.

## Quick start

```bash
# 1. Clone and enter the repository
git clone <repo-url> && cd desi-w0wa-svd

# 2. Install Python dependencies
pip install -e .

# 3. Download external data (~10 GB, requires wget)
bash scripts/download_data.sh

# 4. Reproduce all figures and numbers
bash scripts/reproduce.sh

# 5. Compile the paper
cd paper && pdflatex main && bibtex main && pdflatex main && pdflatex main
```

If you already have `data/figure_data.npz` and `data/paper_numbers.json`
(committed in this repository), you can skip steps 3--4 and go directly to
step 3b:

```bash
# 3b. Generate figures from precomputed data (< 5 seconds)
python scripts/make_all_figures.py
```

## Reproduction pipeline

The pipeline has three steps. Steps 1 and 2 require the downloaded chain data
(~10 GB). Step 3 uses only the precomputed files committed in this repository.

### Step 1: Precompute figure data (~2 min)

```bash
python scripts/precompute_figure_data.py
```

Loads all MCMC chains and external data. Computes every expensive array: chain
predictions at each BAO/SN observable, whitened matrices, SVD decompositions,
w0-wa grid evaluations, extension chain projections. Saves everything to
`data/figure_data.npz`. This is the only script that loops over chain samples
computing cosmological distances.

### Step 2: Compute paper numbers (~2 min)

```bash
python scripts/compute_paper_numbers.py
```

Loads chains and `data/figure_data.npz`. Computes every number quoted in the
paper: tensions, R^2 values, beta-vector coefficients, chi-squared values,
curvature regression, extension sigma_res thresholds. Saves to
`data/paper_numbers.json`.

### Step 3: Generate figures (< 5 sec)

```bash
python scripts/make_all_figures.py
```

Reads `data/figure_data.npz` and `data/paper_numbers.json`. Produces all 17
figure PDFs in `paper/figures/`. No chain loading, no cosmology evaluations.
This is a marimo notebook that also runs as plain Python.

### Orchestration

```bash
bash scripts/reproduce.sh    # Runs all three steps and verifies 17 PDFs
```

## Number provenance

Every number quoted in the paper text is stored in `data/paper_numbers.json`
with a structured key that traces back to the computation.

To trace any number:

1. Find the `\dataref{key}{value}` annotation in the LaTeX source
   (e.g., `\dataref{bao_c0_tension}{+2.2}`)
2. Look up the key in `data/paper_numbers.json`
3. Search for that key in `scripts/compute_paper_numbers.py` to see the
   exact computation

Example:

```
Paper text:    "BAO shows a +2.2 sigma tension"
LaTeX source:  \dataref{bao_c0_tension}{+2.2}
JSON:          {"bao_c0_tension": 2.18, ...}
Script:        tension = (a0 - mean(c0)) / sqrt(1 + var(c0))
```

The `paper_numbers.json` file is committed so that the paper can be compiled
without re-running the computation pipeline.

## Claim structure

The `structure/` directory contains YAML registries that map every scientific
claim to its evidence:

- **`claims.yaml`** -- Each claim has an ID, statement, evidence pointers
  (figure labels, JSON keys, script locations), and dependency links to other
  claims.

- **`figures.yaml`** -- Each of the 17 figures is registered with its source
  script, the marimo cell that produces it, the paper section where it appears,
  and which claims it supports.

- **`logic.yaml`** -- The argument skeleton: what each section establishes,
  what it depends on, and how it connects to the broader research landscape.

- **`claim_dashboard.html`** -- An interactive HTML viewer that renders the
  claim-evidence graph. Open in a browser to explore the provenance chain from
  any claim down to the code and data that support it.

## How this was made

This paper was written entirely using [Claude Code](https://claude.ai/code)
(Anthropic's AI coding agent) in collaboration with Matias Zaldarriaga. The
development process -- including data analysis, figure production, LaTeX
writing, and claim verification -- was carried out through iterative
human-AI sessions.

The claim annotation system (`paperclaims.sty` and the YAML registries) was
developed as part of this project to enable systematic verification that every
scientific claim traces to code and data.
